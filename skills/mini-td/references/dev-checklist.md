# 开发检查清单

## 改动前

1. 明确需求属于哪一层：`core` / `platform` / `render` / `config`。
2. 确认是否涉及多端能力；如涉及，先设计统一接口再落地平台实现。
3. 评估是否需要同步文档和注释。

## 改动中

1. 小步提交，避免一次性大改。
2. 注释使用中文，且只写必要解释。
3. 保持已有命名风格与目录风格一致。

## 改动后

1. 自测运行（至少微信开发者工具可编译）。
2. 检查是否出现跨层耦合（如 `core` 调用平台 API）。
3. 更新文档（README 或 skill references）。
4. 使用规范 commit message。
