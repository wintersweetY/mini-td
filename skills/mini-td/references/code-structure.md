# 代码结构与职责

## 顶层结构

- `game.js`：小游戏运行入口。
- `game.json`：小游戏运行配置。
- `project.config.json`：微信开发者工具项目配置。

## js 目录职责

- `js/main.js`
  - 负责组装：平台适配、画布初始化、游戏实例、渲染器、主循环。
- `js/core`
  - 平台无关的核心逻辑。
  - 示例：波次、经济、敌人工厂、战斗结算、路径推进。
- `js/platform`
  - 平台差异封装（微信、抖音、fallback）。
  - 目标：对上层暴露统一接口，避免业务层感知平台细节。
- `js/render`
  - 渲染与视觉输出，不放规则逻辑。
- `js/config`
  - 塔、敌人、关卡等静态配置数据。
- `js/runtime`
  - 运行时常量和运行辅助。
- `js/utils`
  - 通用工具函数。

## 分层边界

1. `core` 不依赖 `wx`、`tt` 全局对象。
2. `render` 读取快照数据，不直接更改核心状态。
3. `platform` 负责能力适配，不承载游戏规则。
